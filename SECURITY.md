# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please email the maintainers instead of using the public issue tracker. This allows us to address the security issue before it becomes public.

## Security Best Practices

### For Users
- Keep dependencies updated
- Use strong passwords
- Enable two-factor authentication where possible
- Review authentication credentials regularly

### For Contributors
- Never commit sensitive data (API keys, passwords, tokens)
- Review code for security vulnerabilities
- Use environment variables for sensitive configuration
- Follow OWASP security guidelines

## Supported Versions

We provide security updates for the latest version. Older versions may not receive security patches.

## Security Measures

- Input validation and sanitization
- SQL injection prevention
- XSS protection
- CSRF token validation
- Rate limiting on sensitive endpoints
- Secure headers configuration

Thank you for helping keep this project secure!
